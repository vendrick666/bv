# Telegram Bot

Простой Telegram-бот для продажи Instagram-аккаунтов с проверкой подписки на канал, мульти-языковой поддержкой и административными командами.

## Описание

Бот поддерживает:
- выбор языка (русский / английский)
- проверку подписки на канал перед доступом к основному меню
- покупку аккаунтов с фильтрацией по году и количеству подписчиков
- автоматическое формирование файла с купленными аккаунтами
- просмотр профиля пользователя, FAQ, отзывы и поддержку
- админ-команды для обновления базы, рассылки, статистики и добавления аккаунтов вручную

## Требования

- Python 3.10+
- Установленные пакеты:
  - `python-telegram-bot`
  - `requests`
  - `nest_asyncio`

## Установка

1. Откройте терминал в папке с `bot.py`.
2. Установите зависимости:

```powershell
python -m pip install python-telegram-bot requests nest_asyncio
```

## Конфигурация

В файле `bot.py` задаются ключевые параметры:

- `TOKEN` — токен вашего Telegram-бота
- `CHANNEL_ID` — ID канала для проверки подписки (формат `@channelname`)
- `ADMIN_PROFILE` — имя администратора для вывода контактов поддержки
- `ADMIN_CHAT_ID` — список ID администраторов для команд управления
- `PAYMENT_METHODS` — способы оплаты и ссылки

Пример:

```python
TOKEN = 'BOT_TOKEN'
CHANNEL_ID = '@telegram-id'
ADMIN_PROFILE = 'Admin'
ADMIN_CHAT_ID = ['7141098383']
```

> Рекомендуется вынести реальные значения в переменные окружения или отдельный конфигурационный файл, чтобы не хранить токен в открытом виде.

## Запуск

```powershell
python bot.py
```

При первом запуске автоматически создаются базы данных:
- `accounts.db`
- `users.db`

## Команды бота

- `/start` — начать диалог с ботом
- `/help` — показать справку
- `/menu` — вернуться в главное меню

### Администраторские команды
- `/update` — обновить базу аккаунтов
- `/stats` — получить статистику бота
- `/broadcast` — отправить рассылку всем пользователям
- `/addaccount` — добавить аккаунт вручную

## Структура данных

Бот использует SQLite для хранения:
- аккаунтов (`accounts.db`, таблица `accounts`)
- проданных аккаунтов (`sold_accounts`)
- пользователей (`users.db`, таблица `users`)

## Пользовательские сценарии

- проверка подписки на канал
- выбор года создания аккаунтов
- выбор диапазона количества подписчиков
- выбор способа оплаты
- загрузка скриншотов чеков и документов
- поддержка отзывов и FAQ

## Примечания

- В коде предусмотрен периодический фоновой таймер очистки блокировок аккаунтов.
- Если бот не получает доступа к каналу или токен некорректен, он логирует ошибки через `logging`.
